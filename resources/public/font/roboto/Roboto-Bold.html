2a. Tooltips

When you hover “Share” or “Heart” buttons you will notice Tooltips don’t work. That because we have to initialize them first.

 

In the script section of our project, immediate below the initialization code of the SideNav, place the following code:


<script>
        $(document).ready(function () {
            $('[data-toggle="tooltip"]').tooltip({
                animation: true,
                delay: {
                    show: 300,
                    hide: 300
                }
            });
        });
    </script>

 

Save the file and refresh the browser. Hover the buttons now. Our Tooltips works as expected.

Fixing the layout

Posted by Michal Szymanski at 24th Feb 16 5:25 pm
Social shares:

The previous lesson

As we mention before there is an issue related to our layout – SideNav overlaps content and Navbar.

The solution is very easy.

 

Because SideNav is equal to 240px we need to add 240px padding-left to the layout elements: <header> <main> and <footer>.

Thanks to this we’ll create an empty space on the left side, which suits perfectly for our SideNav.

 

Add the following code to style.css:


main,
footer,
.double-navbar {
    padding-left: 240px;
}

main {
    padding-top: 120px;
}

@media only screen and (max-width: 760px) {
    main {
        padding-top: 60px;
    }
}

@media only screen and (max-width: 992px) {
    main,
    footer,
    .double-navbar {
        padding-left: 0;
    }
}

 

That’s enough to fix the layout.

You can see we remove a padding-left for screens smaller than 992px. We have to because otherwise SideNav would make our website illegible on mobile. Now users can slide out a hidden SideNav by clicking the button.

 

Unfortunately now occurs another issue – titles of the sections adjacent to the SideNav. There is no margin and it doesn’t look good.

This is an opportunity to use “main-wrapper“.

 

Paste the following code to style.css:


/* Main wrapper */

.main-wrapper {
    margin-top: 50px;
}


/* Extra Small Devices, Phones */

@media only screen and (max-width: 480px) {
    .main-wrapper {
        margin-left: 0;
        margin-right: 0;
    }
}


/* Small Devices, Tablets */

@media only screen and (min-width: 768px) {
    .main-wrapper {
        margin-left: 3%;
        margin-right: 3%;
    }
}


/* Medium Devices, Desktops */

@media only screen and (max-width: 992px) {
    .main-wrapper {
        margin-left: 4%;
        margin-right: 4%;
    }
}


/* Large Devices, Wide Screens */

@media only screen and (min-width: 1400px) {
    .main-wrapper {
        margin-left: 5%;
        margin-right: 5%;
    }
}

 

Save the file and refresh your browser. You will notice our content now has a nice margin.

When you resize the browser you can see the margin changes. That’s because we’ve used relative units (percentages) and that makes a margin flexible.

Additionally, we use a media queries (“@media only screen[..]” ) to make a layout even more responsive.

2. Consistency in the color pattern

Color consistency is one of that small, underestimated things, which often decide if a website looks really professional.

 

Of course, in our current project, we want do make everything perfect, including colors. But if you look at our website right now you will notice that each element (navbar, buttons, footer etc. ) is a completely different color.

We have to change it. We’ll use the color of our Navbar (#405C84) as a pattern.

 

Add the following code to the style.css:


.pt-color {
  background-color: #405C84 !important;
}

.st-color {
  background-color: #0E274D !important;
}

 

With the code above we set a 2 base color for our templates: pt-color (primary template color) and st-color (secondary template color).

These are 2 shades of the same color and they look really good together.

We’ve also used a !important word to avoid a potential conflict with other CSS classes (for examples 2 different values of the same property).

It says to the browser: “If you encounter a conflict always choose an option with the word !important.

 

Be very careful when using !important.

Actually, if you don’t have a lot of experience is better to avoid using this. It can cause bugs which are hard to discover.

But this time, is fine, trust me.

 

Now let’s see our new colors in action. In the Footer replace the class “blue” with the class “pt-color”:


<!--Section: Footer-->
    <footer class="page-footer pt-color" id="sec-footer">

 

Now the Footer color is consistent with the color of our Navbar. We have to do the same with the rest of elements, like buttons.

 

Buttons can take 3 different states: inactive, hover and focus. That means they can have a slightly different colors when you hover or click them.

Add a code below to your style.css:


.btn-ptc {
  background-color: #405C84 !important;
}

.btn-ptc:hover {
  background-color: #5174a6 !important;
}

.btn-ptc:focus {
  background-color: #5174a6 !important;
}

.btn-stc {
  background-color: #0E274D !important;
}

.btn-stc:hover {
  background-color: #163d78 !important;
}

.btn-stc:focus {
  background-color: #163d78 !important;
}

 

All the colors above are just a shadows of our 2 main colors (pt-color and st-color).

Save the file and refresh your browser. You can see the rectangular buttons in the Intro have already a value and color pt-color.

That’s because in the previous lessons we’ve added to them a class .btn-ptc .

Now hover them. You will notice they brighten up when you hover the mouse. That’s thanks to so-called pseudo-selector :hover which we’ve used before.

 

Now let’s take care of floating (round) buttons and Carousel indicators.

Add a following code to the style.css:


.btn-floating,
.carousel-indicators li {
  background-color: #0E274D !important;
}

 

You will notice all the floating buttons have changed colors. All apart from the Carousel buttons. That’s because they already have a color classes “grey darken-4“.

Remove these classes and everything should work as expected

1.  Fixing responsiveness of the Intro.

Before we go any further we need to correct a few things related to the responsiveness of our Intro.

When you resize your browser to the mobile size you will see the Navbar and section below Intro overlap it. That’s because construction which we’ve built for centering our content works fine on the desktop, but it has to be improved for mobiles and tablets.

It easy to fix with an appropriate media query. Add the following code to your style.css:


@media only screen and (max-width: 768px) {
  .full-bg-img, .view {
    height: auto;
    position: relative;
    text-align: center;
  }
  .full-bg-img .verticalcenter, .view .verticalcenter {
    top: none;
    margin-top: 40px;
    -webkit-transform: none;
    -o-transform: none;
    transform: none;
  }
  .full-bg-img .container, .view .container {
    padding-top: 30px;
  }
}

s

Thanks to that on all the screens smaller than 768px our Intro will change a few of its properties.  Save the file and refresh the browser.

Now when you resize your browser, the content within the Intro is still perfectly displayed. Well done.

 

2. Navbar transparency

There are 2 problems with our Navbar.

Firstly, when we scroll the Navbar become white and that makes our links invisible.

Secondly, on the mobile, the Navbar stays transparent even after scrolling down. It’s not the desired behavior because it will overlap the content of our website.

 

To fix the first problem add the following code to your style.css:


.top-nav-collapse {
  background-color: #405C84;
}

.top-nav-collapse .nav li a {
  color: #fff;
}

 

.top-nav-collapse is a special class which is added (via JavaScript) to the Navbar immediately after scrolling. So if we set a background color for it, we’ll achieve the desired result.

 

To fix the second issue we need to use again a media query. Add a code below to your style.css:


@media only screen and (max-width: 768px) {
  .navbar {
    background-color: #405C84;
  }
  .navbar .navbar-nav > li > a {
    padding-top: 10px;
    padding-bottom: 10px;
    color: #fff;
  }
}

 

We’ve set a background color for the mobile version of our Navbar so it won’t be transparent on any device smaller than 768px. Transparent Navbar looks good on big screens but on mobile can be problematic for users.

 

Now our Intro is ready and we can start to play with the cards and carousel.
